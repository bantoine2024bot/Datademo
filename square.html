What happens when you visit a website?

High-level flow of a page load.

1) You type a URL

Your browser resolves the domain, opens a connection, and requests a document (usually index.html).

Client (your browser)

2) Server responds

A web server sends back HTML. The browser starts parsing it top-to-bottom.

Server

3) Browser renders

While parsing HTML, the browser discovers CSS & JS files, downloads them, builds the DOM & CSSOM, and paints pixels.

Render engine


Browser              Server
   |  GET /index.html  |   → Request
   |------------------->|
   |  200 OK (HTML)     |   ← Response
   |<-------------------|
   | parse HTML → fetch linked CSS/JS/Images
   | build DOM/CSSOM → layout → paint

Tip: Open F12 → Network tab, then click any button below that makes a request to watch it happen.

HTML — Structure & Meaning

HTML describes what is on the page: headings, paragraphs, links, lists, images, and more. Tags (like <h1>) give content semantic meaning.

Example Markup

<article>
  <h1>A Blog Post</h1>
  <p>Hello, world!</p>
  <a href="/about">About me</a>
</article>

JavaScript — Behavior

JavaScript lets pages respond to user actions, update the DOM, and talk to servers.

Show sample JS 
// Find a DOM node
const btn = document.querySelector('#count-btn')

// Update text when clicked
let n = 0
btn.addEventListener('click', () => {
  n++
  btn.textContent = `Clicked ${n} times`
})
